## 1. GitHub Release 简介

### 标题

`Qoreon v1 Candidate`

### 正文

Qoreon 是连接人类意图与 AI 执行的核心系统。

它让多个 AI Agent 可以被组织、协同和持续优化。  
你不再直接用 AI，而是管理一个 AI 团队。

这个候选版本包含：

- 本地运行的多 Agent 协同控制层
- 可视化任务看板与运行页
- 最小示例项目
- 通道、Agent、任务、技能的种子包
- 面向外部 AI 的初始化入口

适合用来做：

- 本地 AI 团队协作试验
- 多 CLI Agent 的统一控制
- 可继续扩展的任务空间模板

## 3. 社交平台首发短文

### 版本 A：产品定位

我们最近把一个内部多 Agent 协同控制层，整理成了一个可公开运行的候选版本：Qoreon。

Qoreon 不是“再包一层 prompt”，而是把多个 AI Agent 组织成一个可协作、可回看、可持续优化的执行系统。

它本地运行，能接入 Codex、Claude Code 等 CLI，并提供统一的任务、通道、种子包和控制层。

### 版本 B：一句话抓手

Qoreon：你不再直接用 AI，而是管理一个 AI 团队。

### 版本 C：技术向

Qoreon is a local-first control layer for multi-agent execution.  
It turns task spaces into visible boards, coordinates CLI agents, and ships with seed packs plus AI bootstrap docs.

## 4. 首批对外介绍长文框架

### 标题

`为什么我们需要 Qoreon：从“用一个 AI”到“管理一个 AI 团队”`

### 结构

1. 问题：单会话 AI 工具很强，但难以组织复杂协作
2. 转折：真正缺的是控制层，而不是更多 prompt
3. 方案：Qoreon 如何把任务、通道、Agent、技能和执行放进一个系统
4. 首发边界：为什么先做本地优先、公开安全的一体包
5. 下一步：远程仓库、更多示例、更多接入器
